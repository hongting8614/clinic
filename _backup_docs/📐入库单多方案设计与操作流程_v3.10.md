## 🎨 设计方案总览

| 方案 | 特点 | 适用场景 | 复杂度 |

| 方案1 | 极简单页式 | 快速录入 | ⭐ |

## 🎯 交互规则总结

### 规则1: 智能展开/收起
```
触发条件                    → 行为
─────────────────────────────────────
新添加药品                  → 自动展开编辑
填写完成                    → 自动收起
点击卡片                    → 切换展开/收起
点击其他卡片                → 当前收起，新的展开
全部展开按钮                → 所有展开
全部收起按钮                → 所有收起
```

### 规则2: 数据验证时机
```
验证时机                    → 验证内容
─────────────────────────────────────
输入框失焦                  → 格式验证
日期选择                    → 逻辑验证（生产<有效）
数量输入                    → 范围验证（>0）
保存草稿                    → 宽松验证（允许未完成）
提交复核                    → 严格验证（必须完整）
```

### 规则3: 自动计算
```
触发条件                    → 计算内容
─────────────────────────────────────
输入数量或单价              → 单行金额
输入有效期                  → 距有效期天数
添加/删除药品               → 合计品种数
修改数量                    → 合计数量
修改金额                    → 合计金额
```

### 规则4: 颜色警示
```
距有效期                    → 颜色
─────────────────────────────────────
< 0 天（已过期）            → 红色 + 禁止入库
≤ 60 天（临期）             → 橙色 + 警告提示
≤ 90 天（近期）             → 黄色 + 注意提示
> 90 天（正常）             → 绿色
```

### 规则5: 操作确认
```
操作类型                    → 是否需要确认
─────────────────────────────────────
删除药品                    → ✓ 需要确认
清空全部                    → ✓ 需要确认
提交复核                    → ✓ 需要确认
保存草稿                    → ✗ 直接保存
收起/展开                   → ✗ 直接操作
输入内容                    → ✗ 直接输入
```
