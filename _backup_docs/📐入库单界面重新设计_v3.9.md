## 🎯 设计分析

### 当前界面问题
1. ❌ 基本信息区域过大，占用太多空间
2. ❌ "暂无药品明细"提示过于突出
3. ❌ 签名区域位置不合理
4. ❌ 缺少快速操作入口
5. ❌ 信息层次不够清晰

### 改进目标
1. ✅ 紧凑显示基本信息
2. ✅ 突出药品明细编辑区
3. ✅ 优化签名流程
4. ✅ 增加快捷操作
5. ✅ 提升视觉层次

## 💡 交互设计

### 1. 添加药品流程
```
点击[📋选择] → 跳转药品列表
              → 选择药品
              → 自动返回
              → 新卡片从下方滑入
              → 自动展开编辑状态

点击[✍️新建] → 弹出新建表单
              → 填写基本信息
              → 确认添加
              → 添加到列表
```

### 2. 卡片交互
```
点击卡片       → 展开/收起
点击[︙]       → 显示更多操作菜单
                 - 复制
                 - 移到顶部
                 - 移到底部
                 - 删除
点击[×]       → 确认删除
左滑卡片       → 显示删除按钮
长按卡片       → 进入拖拽排序模式
```

### 3. 表单验证
```
实时验证:
- 批号填写时: 检查是否重复
- 日期选择时: 检查逻辑关系
- 数量输入时: 检查范围
- 金额自动计算

提交验证:
- 标记错误字段（红色边框）
- 展开有错误的卡片
- 滚动到第一个错误位置
- Toast提示具体错误
```

### 4. 签名流程
```
未签名状态:
┌──────────────────────┐
│ ✍️ 操作员签名 *      │
│        [点击签名]    │
└──────────────────────┘

点击后:
┌──────────────────────┐
│  手写签名弹窗        │
│  ┌────────────────┐  │
│  │                │  │
│  │   签名区域     │  │
│  │                │  │
│  └────────────────┘  │
│  [重签] [确定]       │
└──────────────────────┘

已签名状态:
┌──────────────────────┐
│ ✍️ 操作员签名        │
│  ✓ 张三 (已签名)    │
│        [重新签名]    │
└──────────────────────┘
```

## 🎯 优化要点

### 空间利用
- ✅ 基本信息3行 → 2行（节省40rpx）
- ✅ 药品卡片默认折叠（节省200rpx/个）
- ✅ 签名单行显示（节省280rpx）
- ✅ 总计节省: 约520-800rpx

### 操作效率
- ✅ 顶部快捷保存按钮
- ✅ 药品明细标题栏直接操作
- ✅ 卡片折叠减少滚动
- ✅ 左滑快速删除

### 视觉层次
```
重要性排序:
1. 药品明细（核心）
2. 合计统计（结果）
3. 签名状态（必要）
4. 基本信息（次要）

视觉权重:
大 → 药品卡片（展开时最大）
中 → 合计卡片、签名区
小 → 基本信息条
```

### 移动友好
- ✅ 按钮高度 ≥ 88rpx
- ✅ 输入框高度 ≥ 70rpx
- ✅ 点击区域 ≥ 88rpx × 88rpx
- ✅ 支持左滑、长按手势
- ✅ 底部按钮避开safe-area
