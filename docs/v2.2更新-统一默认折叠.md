# v2.2 更新说明 - 统一默认折叠

## 📅 更新时间
2024年11月

## 🎯 更新内容

### 核心变化
**所有药材（包括新药材）选择后都默认折叠状态**

---

## 📊 对比说明

### ❌ v2.1 版本
```
已有药材（档案选择/扫码）：showDetail: false ✅ 默认折叠
新药材（手动输入）：        showDetail: true  ⚠️ 默认展开
```

**问题：**
- 新药材自动展开，占用较多空间
- 批量添加时界面不够简洁
- 体验不统一

---

### ✅ v2.2 版本
```
所有药材（无论来源）：showDetail: false ✅ 统一默认折叠
```

**优势：**
- ✅ 界面更简洁统一
- ✅ 批量录入效率更高
- ✅ 一屏显示更多药材
- ✅ 操作体验一致

---

## 🔧 技术修改

### 1. 数据初始化
```javascript
// 修改前（v2.1）
addEmptyDrug(barcode) {
  this.drugList.push({
    // ...
    isNewDrug: true,
    showDetail: true  // ❌ 新药材默认展开
  })
}

// 修改后（v2.2）
addEmptyDrug(barcode) {
  this.drugList.push({
    // ...
    isNewDrug: true,
    showDetail: false  // ✅ 新药材也默认折叠
  })
}
```

### 2. 模板逻辑
```vue
<!-- 修改前（v2.1） -->
<!-- 展开/收起按钮（仅已有药材） -->
<view v-if="!item.isNewDrug" class="toggle-detail-btn" @click="toggleDetail(index)">
  <text>{{ item.showDetail ? '收起详情' : '展开详情' }}</text>
</view>

<!-- 详细信息（新药材自动显示） -->
<view v-if="item.showDetail || item.isNewDrug" class="drug-info">
  <!-- ... -->
</view>


<!-- 修改后（v2.2） -->
<!-- 展开/收起按钮（所有药材都有） -->
<view class="toggle-detail-btn" @click="toggleDetail(index)">
  <text>{{ item.showDetail ? '收起详情' : '展开详情' }}</text>
</view>

<!-- 详细信息（统一折叠逻辑） -->
<view v-if="item.showDetail" class="drug-info">
  <!-- ... -->
</view>
```

## 🎯 使用流程

### 场景：手动添加新药材

**v2.1 流程：**
```
扫码失败 → 选择手动输入 → 新药材卡片自动展开（占用大量空间）
→ 填写药材详细信息 → 填写批号/日期/数量 → 提交
```

**v2.2 流程（优化后）：**
```
扫码失败 → 选择手动输入 → 新药材卡片默认折叠（简洁）
→ 点击「展开详情」→ 填写药材详细信息 
→ 填写批号/日期/数量 → 提交
```

**优势：**
- ✅ 批量添加多个新药材时，界面更简洁
- ✅ 一屏可显示更多药材
- ✅ 只在需要时展开详情

## 💡 使用建议

### ✅ 推荐做法

1. **批量添加多个新药材**
   - 保持所有药材折叠状态
   - 先添加所有药材到列表
   - 逐个展开填写详细信息
   - 界面简洁，效率更高

2. **单个新药材详细填写**
   - 添加药材后，点击「展开详情」
   - 认真填写所有详细信息
   - 填写批号、日期、数量
   - 提交创建档案

### 📝 操作提示

**新药材卡片标识：**
```
💊 [新药材] 请输入药材名称
```
- 橙红色「新药材」标签
- 药材名称可编辑
- 其他字段需展开后填写
